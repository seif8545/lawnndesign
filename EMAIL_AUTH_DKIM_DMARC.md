# Lawnn — Email Authentication (DKIM / SPF / DMARC) for `lawnndesign.com`

Goal: make Brevo's acceptance emails from `info@lawnndesign.com` pass authentication so
Gmail / Yahoo / Outlook deliver them to the inbox instead of spam.

There are three records. You add all three at whoever hosts DNS for `lawnndesign.com`
(likely Cloudflare, since the site is on Cloudflare Pages — if so, add them in the same
Cloudflare DNS panel).

> **Important — one honest limitation:** the **DKIM** values are generated by Brevo and are
> unique to your account. I can't invent them (and shouldn't — wrong values just fail
> silently). You copy them from the Brevo dashboard exactly as shown. The **DMARC** record
> below is standard and I've written it out in full — you can paste it verbatim.
>
> If you'd rather not copy them by hand, I can open the Brevo dashboard in your browser and
> read the exact DKIM records for you — just say so.

---

## 1. DKIM + domain verification — copy from Brevo

DKIM is the signature that proves the email really came from your domain. Brevo issues the
records; you publish them in DNS.

**In Brevo:**

1. Go to **Senders, Domains & Dedicated IPs → Domains** (top-right menu → your account).
2. Find `lawnndesign.com` (the handoff says it's already added/verified) and click
   **Authenticate** / **Configure**.
3. Brevo shows a short list of records to add — typically:
   - one or more **DKIM** records (a `CNAME` or `TXT` on a host like `brevo._domainkey` /
     `mail._domainkey` — the exact selector and value are shown there),
   - a **domain-verification** `TXT` record (a `brevo-code:…` string),
   - and possibly an **SPF** entry (see §2).

**Copy each one exactly** — host/name, type, and value — into your DNS host. Do not retype
from memory; a single wrong character makes DKIM fail.

> **Cloudflare users:** add these as normal DNS records and make sure each is set to
> **"DNS only" (grey cloud, not proxied)**. Proxying (orange cloud) breaks mail records.

After the records propagate (minutes to a couple of hours), click **Verify / Authenticate**
in Brevo. You want it to show DKIM = **authenticated**, not just "verified."

---

## 2. SPF — one record only

SPF lists who's allowed to send mail for your domain.

- If Brevo's authentication screen gives you an SPF value to add, add it.
- **Critical rule:** a domain may have **only one** SPF `TXT` record. If you already have an
  SPF record (e.g. from Google Workspace / another sender), **merge** Brevo's part into the
  existing one — do **not** create a second `v=spf1` record. Two SPF records = SPF fails
  entirely.
- A merged example looks like: `v=spf1 include:spf.google.com include:<brevo's-include> ~all`
  (use whatever include value Brevo actually shows — don't guess it).

If you have no existing SPF record and Brevo doesn't require one for its modern setup, that's
fine — DKIM + DMARC alignment is what matters most for deliverability.

---

## 3. DMARC — paste this one exactly

DMARC tells receivers what to do with mail that fails DKIM/SPF, and sends you reports. Add a
`TXT` record:

| Field | Value |
|---|---|
| **Type** | `TXT` |
| **Name / Host** | `_dmarc` &nbsp;(some hosts want the full `_dmarc.lawnndesign.com`) |
| **Value** | `v=DMARC1; p=none; rua=mailto:info@lawnndesign.com; fo=1; adkim=r; aspf=r; pct=100` |

What the tags mean:

- `p=none` — **monitor mode.** Nothing gets blocked yet; you just collect reports. Start here.
- `rua=mailto:info@lawnndesign.com` — where aggregate reports are sent.
- `fo=1` — send a failure report if either DKIM or SPF fails.
- `adkim=r` / `aspf=r` — relaxed alignment (the normal, forgiving setting).
- `pct=100` — policy applies to all mail.

**Rollout (don't skip straight to blocking):**

1. Launch with `p=none` and send your first batch of acceptance emails.
2. Watch for ~1–2 weeks. Confirm in Brevo (and any DMARC reports) that your mail passes DKIM.
3. Then tighten to `p=quarantine` (failing mail → spam), and later `p=reject` (failing mail
   dropped) once you're confident nothing legitimate fails.

## 4. Verify it worked

- **Brevo dashboard:** the domain shows DKIM authenticated (green).
- **Send yourself a test** to a Gmail address. In Gmail, open the message → **⋮ → Show
  original**. You want **SPF: PASS**, **DKIM: PASS**, **DMARC: PASS**.
- Only after DKIM shows PASS should you send a large batch. (Brevo free tier = **300
  emails/day** — split big lists across days.)

## Quick checklist

- [ ] Brevo → Domains → Authenticate `lawnndesign.com`; copy the DKIM + verification records.
- [ ] Add those records at your DNS host (Cloudflare = "DNS only" / grey cloud).
- [ ] Add/merge SPF only if Brevo provides one — never two SPF records.
- [ ] Add the `_dmarc` TXT record above (`p=none` to start).
- [ ] Back in Brevo, click Verify/Authenticate until DKIM shows authenticated.
- [ ] Send a test to Gmail → "Show original" → SPF/DKIM/DMARC all PASS.
- [ ] Then run the acceptance-email batch (≤300/day).
- [ ] After 1–2 clean weeks, raise DMARC to `p=quarantine`, then `p=reject`.
